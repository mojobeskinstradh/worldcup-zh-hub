# worldcup-zh-hub

本项目用于归档和发布多个独立 HTML 页面。这些页面内容不局限于某一特定网站或域名，而是作为一个通用的页面集合仓库，方便查阅、分发与维护。

## 项目简介

`worldcup-zh-hub` 是一个以中文内容为主的 HTML 页面归档仓库。所有页面均为独立静态文件，不依赖后端服务或数据库。仓库的主要目的是集中存放各类信息页面，并利用 GitHub Pages 等功能进行静态发布。

## 目录说明

```
.
├── index.html          # 主页 / 导航页
├── pages/              # 存放各独立 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 公共资源目录（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **`index.html`**：仓库的入口页面，提供页面列表或简要导航。
- **`pages/`**：存放所有独立 HTML 页面，文件名应具有描述性。
- **`assets/`**：存放页面共享的样式、脚本及图片文件，便于统一管理。

## 页面归档说明

- 所有页面均为纯静态 HTML，可直接在浏览器中打开。
- 页面内容以中文为主，不针对任何特定第三方网站或域名。
- 每个页面应尽量自包含，但可引用 `assets/` 下的公共资源。
- 页面之间不应有硬编码的绝对路径依赖，推荐使用相对路径。

## 维护说明

- 欢迎提交 Pull Request 添加新的页面或改进现有页面。
- 新增页面请放入 `pages/` 目录，并在 `index.html` 中添加链接。
- 请确保页面无恶意代码、无广告弹窗、无侵权内容。
- 资源文件请放入对应子目录，避免直接在根目录堆积文件。
- 如有重大结构调整，请先通过 Issue 讨论。
